angular.json: CLI configuration file for configuring workspaces, build targets, configurations, styles/scripts assets, and performance budgets.
tsconfig.json: Root TypeScript compiler configuration that specifies base options and references project-specific configurations.
tsconfig.app.json: TypeScript compiler configuration tailored specifically for building the application, including the entry point file and ts files.
package.json: Defines package metadata, npm script shortcuts, and lists workspace-wide runtime and development dependencies.
src/main.ts: The main entry point of the application that bootstraps the root component (App) using standalone mode.
src/app/app.config.ts: Configuration file for specifying application-wide providers, routes, and dependency injection services.
src/app/app.ts: The root component (App) of the application, representing the shell and primary starting point of the layout.
src/index.html: The single page HTML entry point containing the '<app-root>' tag where Angular injects the application.

Budgets Configuration:
In 'angular.json', the budgets configuration under 'architect > build > configurations > production' defines size thresholds for the bundle sizes.
- For initial bundle: it sets a warning threshold at 500kB and an error threshold at 1MB to prevent sending excessively large bundles to users.
- For component style bundle: it sets a warning threshold at 4kB and an error threshold at 8kB to keep component-specific CSS small.
If these thresholds are exceeded during a production build, the build emits a warning or fails with an error respectively.
